Common Vulnerability Types

Part 1: Software Flaws

💡 Introduction:

Security vulnerabilities are unintentional flaws or weaknesses in software and hardware. If left unpatched, these flaws can be exploited by attackers to bypass security measures and gain unauthorized control over systems.

Common Weaknesses

  • SQL Injection
    • Targets databases by inserting malicious SQL queries.
    • Can lead to unauthorized viewing or deletion of data.
  • Cross-Site Scripting (XSS)
    • Injects malicious scripts into trusted websites.
    • Often used to steal session cookies or user personal data.
  • Weak Authentication
    • Includes weak passwords and lack of multi-factor authentication.
    • Makes it easy for attackers to perform brute-force attacks.

"Finding the Gap: Attackers exploit unpatched flaws and 'cracks' in software to bypass security measures.
